HTML Versus XHTML

XHTML, que significa EXtensible HyperText Markup Language, é uma versão mais rigorosa e baseada em XML do HTML. É essencialmente HTML definido como uma aplicação XML e é suportado por todos os principais navegadores.

Por que XHTML?

A necessidade de XHTML surge da busca por uma linguagem de marcação que seja mais extensível e flexível, especialmente em relação à compatibilidade com outros formatos de dados, como o XML. Ao contrário do HTML, onde os navegadores ignoram erros e tentam exibir o conteúdo mesmo que haja falhas na marcação, o XHTML adota um tratamento de erros mais estrito, exigindo que os documentos estejam sempre "bem formados".

Principais Diferenças do XHTML em Relação ao HTML

  • Declaração <!DOCTYPE>: obrigatória em todos os documentos XHTML.
  • Elementos obrigatórios: <html>, <head>, <title> e <body> devem estar presentes.
  • Aninhamento de Elementos: Os elementos devem ser corretamente aninhados.
  • Fechamento de Elementos: Todos os elementos devem ser fechados.
  • Minúsculas: Todos os nomes de elementos e atributos devem estar em letras minúsculas.
  • Valores de Atributos: Devem ser sempre entre aspas.
  • Minimização de Atributos: É proibida.

Exemplo de Estrutura XHTML

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" 
"http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
<head>
  <title>Título do Documento</title>
</head>
<body>

  conteúdo aqui...

</body>
</html>

Regras de Sintaxe em XHTML

1. Aninhamento Correto

Certo: <b><i>Texto em itálico</i></b>

Errado: <b><i>Texto em itálico</b></i>

2. Fechamento de Elementos

Certo: <p>Este é um parágrafo</p>

Errado: <p>Este é um parágrafo

3. Elementos Vazios

Certo: <br />

Errado: <br>

4. Letras Minúsculas

Certo: <body><p>Este é um parágrafo</p></body>

Errado: <BODY><P>Este é um parágrafo</P></BODY>

5. Nomes e Valores de Atributos

Certo: <a href="https://www.exemplo.com">Visite nosso tutorial</a>

Errado: <a HREF=https://www.exemplo.com>Visite nosso tutorial</a>

Conclusão

A rigidez do XHTML visa garantir a consistência e a precisão dos documentos, promovendo uma prática de codificação que evita erros comuns encontrados na construção de páginas HTML tradicionais. Essa abordagem é particularmente útil em contextos onde a interoperabilidade entre diferentes sistemas e formatos de dados é essencial.
