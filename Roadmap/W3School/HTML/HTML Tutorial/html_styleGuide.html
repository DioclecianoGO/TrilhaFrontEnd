Guia de Estilo HTML

Declaração do Tipo de Documento: O primeiro elemento de um documento HTML deve ser a declaração do tipo de documento.

<!DOCTYPE html>

Utilizar Nomes de Elementos em Minúsculas: Embora o HTML permita letras maiúsculas, recomendamos o uso de letras minúsculas para melhor legibilidade.

<body></body>

<BODY></BODY>

Fechar todos os elementos HTML: Fechar todos os elementos é uma boa prática, mesmo que alguns elementos não exijam.

<section>
  <p>Este é um parágrafo.</p>
  <p>Este é outro parágrafo.</p>
</section>

<section>
  <p>Este é um parágrafo.
  <p>Este é outro parágrafo.
</section>

Utilizar Nomes de Atributos em Minúsculas: Seguindo a mesma lógica dos nomes de elementos, use atributos em minúsculas para maior consistência.

<a href="https://exemplo.com">Visite o Exemplo</a>

<a HREF="https://exemplo.com">Visite o Exemplo</a>

Citar Valores de Atributos: Sempre cite os valores dos atributos, especialmente se houver espaços no valor.

<table class="tabela-bonita">

<table class=tabela-bonita>

Especificar alt, width e height para Imagens: Para evitar alterações no layout e garantir acessibilidade, sempre forneça esses atributos nas imagens.

Evitar Espaços ao Redor de Sinais de Igual: Manter o código limpo e legível sem adicionar espaços desnecessários ao redor de sinais de igual.

<link rel="stylesheet" href="styles.css">

<link rel = "stylesheet" href = "styles.css">

Evitar Linhas de Código Longas: Tente quebrar o código em várias linhas para evitar rolagem horizontal.

Quebras de Linha e Indentação: Use linhas em branco para separar blocos lógicos de código, e indente com dois espaços, nunca com tabs.

<body>
  <h1>Cidades Famosas</h1>
  <h2>Tóquio</h2>
  <p>Tóquio é a capital do Japão.</p>
</body>

Exemplo de Tabela:

Nome Descrição
A Descrição do A
B Descrição do B

Exemplo de Lista:

  • London
  • Paris
  • Tokyo

Nunca Pule o Elemento <title>: O título da página é importante para SEO e deve ser sempre incluído.

<title>Guia de Estilo HTML</title>

Incluir <html> e <body>: Essas tags são opcionais, mas recomendadas para evitar problemas com navegadores antigos e softwares XML/DOM.

<html>
<body>
</body>
</html>

Comentários: Comentários curtos podem ser escritos em uma linha, enquanto comentários longos devem ser quebrados em várias linhas e indentados.

<!-- Comentário curto -->
<!--

  Comentário longo

  Comentário longo

-->

Utilizar Nomes de Arquivos em Minúsculas: Nomes de arquivos devem ser sempre em minúsculas para evitar erros em servidores sensíveis a maiúsculas.

Meta Data: Incluir o charset e o atributo de idioma para garantir a interpretação correta da página e a otimização para mecanismos de busca.


<meta charset="UTF-8">

<html lang="pt-br">

Configurar a Viewport: A tag <meta name="viewport"> garante que a página seja exibida corretamente em diferentes dispositivos.

<meta name="viewport" content="width=device-width, initial-scale=1.0">
